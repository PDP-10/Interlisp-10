Subject: Storage lies about # litatom pages left -- should give # litatoms left?

Lisp Version: 25-Apr-84 20':38':55

Description: When \AtomFrLst was such that there were only 384 atoms left, STORAGE said that there were "7 pages" of atom space left.  Given 128 atoms per "page", it should have said  "3 pages".   I''d prefer, however, that the litatom "space" description simply told how many atoms were left (i.e., how many more litatoms you can  create before falling into raid death).

Workaround: 

Test Case: (PROGN  (until (IGEQ \AtomFrLst 32384) do (GENSYM))  (STORAGE))
