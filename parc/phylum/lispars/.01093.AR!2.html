Subject: Typo in HASHOVERFLOW in MACHINEINDEPENDENT

Lisp Version:  9-May-84 09':41':34

Description: HASHOVERFLOW was rehashing into a hash array created using HARRAY instead of HASHARRAY.  Under the "new" hash array regime, this has the effect of giving the new array the overflow property ERROR, regardless of its previous setting.  Thus, a given HARRAYP would only overflow once successfully; the next time would cause a HASH ARRAY FULL error.
