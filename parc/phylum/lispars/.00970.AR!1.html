Subject: Can get stack overflow in early phase of loadup if error prints

Lisp Version: 

Description: '
A few weeks ago, I was doing a loadup, and got a stack overflow during the phase where INIT.SYSOUT is loading various packages.  I got Richard Burton to look at it, and he decided that it was a long-standing mis-feature':  errors at this early phase cause a call to \TTYOUTCHARFN, which isn''t defined yet, which causes an error, etc...  In this case, the first error was caused by \LEAF.TIMEOUT.  I don''t know what could be done about it, but it might as well be in the AR database.
