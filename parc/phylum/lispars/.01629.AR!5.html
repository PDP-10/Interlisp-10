Looking at the compiled code for CHAT, we think we''ve located the bug.   Refer to CHAT.HANDLECHARACTER at instruction #262 of the compiled listing in INSPECTCODE.  This calls CHAT.ADDCHAR which moves text over one char. width in INSERT-DELETE mode, but doesn''t print the character.  Either CHAT.ADDCHAR should print the character or the RETURN in line #265 of CHAT.HANDLECHARACTER should be removed.'
'
What is happening in Emacs is that Emacs sends the character'
to go into Insert/delete mode, sends a blank, exits I/D mode,'
sends a backspace, and finally the character that was'
supposed to be inserted.  CHAT was never spacing over for'
the blank character because it was never printing it.'
'
Could you send us a patch for this due to the importance of'
this editor for our faculty & staff?   How about an additional'
patch mapping the BACKARROW key to a DEL during CHAT mode?'
'
Judy'
