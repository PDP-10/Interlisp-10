Subject: needless SHAPEW hiccup in MOVEATTACHEDWINDOWTOPLACE

Lisp Version: 14-May-84 20':07':29

Description: '
When a window is attached in JUSTIFY mode, and the window passed to ATTACHWINDOW is already exactly the right size and in the right position, MOVEATTACHEDWINDOWTOPLACE still calls SHAPEW to put the window into its place.  Instead of being a no-op, this closes and opens/redisplays the window, a rather annoying hiccup in places like the backtrace frame window.'
'
Just offhand, it looks like it would be a pain to fix this in ATTACHWINDOW.  So how about having SHAPEW treat request to shape to the same region to be a no-op?  A nice, but less important addition, would be to have it treat a shaping to the same shape but different position to be a MOVEW..
