Symptoms':  On a D0, your color board appears completely dead.  Color calls return values, but the color screen shows only thin vertical stripes of some random color.  (EQ (LRSH (\DEVICE.INPUT 80) 8) 175) returns NIL.'
'
Solution':  The microcode file <Lisp>Current>X3DolphinLispMC.EB *does not work* with color.  Use <Lisp>Current>DolphinLispMC.EB instead.'
'
Recommendation':  (Lispsupport) This ought to be fixed or advertised.'

Glad you found out what the problem was!'
'
The X3DolphinLispMC.EB microcode is intended to be used only if you are trying to be a gateway between 3 and 10 megabit nets.  Having both 3 and 10 ethernet microcode support means that there isn''t room for either color or floating point microcode.  The future documentation will be clearer on this point.'
