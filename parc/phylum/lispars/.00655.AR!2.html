OPENTEXTSTREAM, when given a window, sets the window''s PROCESS prop to be that of the caller, even though there isn''t "a process behind the window".  This is wrong--should only be done by TEDIT.  Two effects': clicking in the window makes some random process the Tty process (typically MOUSE, e.g., when called by Lafite''s Display command or background ShowAR); and clicking in the window pops up a menu that makes it look like there IS a process there.  Ideally, clicking in the window should have NO effect, except to bring the window to the top, and do copy selection if shift is down.'
